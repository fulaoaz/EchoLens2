## 快速开始

### 什么是 EchoLens？

EchoLens 2.0 是一个**电商舆情智能体仿真 + 数据预测决策平台**，帮助您：

- 📊 **真实数据采集**：自动从京东、淘宝、微博、小红书等平台采集商品和舆情数据
- 🤖 **智能体仿真**：使用 OASIS 框架模拟百万级用户的社交媒体行为
- 📈 **预测分析**：基于时序预测和因果推断，预测商品销量和舆情趋势
- 🎯 **决策支持**：综合多维度数据，生成可追溯的决策建议
- 📄 **分析报告**：自动生成包含证据链的专业分析报告

### 5 分钟上手

#### 1. 创建项目

点击右上角 **"新建项目"** 按钮，填写项目信息：

- **项目名称**：例如 "iPhone 15 舆情分析"
- **项目描述**：简要说明分析目标

#### 2. 上传材料

在项目详情页，上传商品相关文档：

- 支持格式：PDF、TXT、Markdown
- 内容示例：商品介绍、营销方案、竞品分析

系统会自动：
- 提取关键信息
- 构建知识图谱
- 生成 Seed Report

#### 3. 数据采集

配置爬虫参数：

- **平台选择**：京东、淘宝、微博、小红书
- **关键词**：商品名称或相关话题
- **采集数量**：建议 100-500 条

点击 **"开始采集"**，等待数据采集完成。

#### 4. 运行仿真

配置仿真参数：

- **智能体数量**：建议 1000-10000
- **仿真轮次**：建议 10-50 轮
- **刺激强度**：正面/负面/中性

点击 **"开始仿真"**，观察实时进度。

#### 5. 查看决策

仿真完成后，进入 **"决策看板"**：

- 查看综合评分
- 点击证据链追溯原始数据
- 查看可靠性等级（强/一般/弱）

#### 6. 生成报告

点击 **"生成报告"**，选择格式：

- **HTML**：可在浏览器中查看
- **PDF**：适合打印和分享
- **Markdown**：适合进一步编辑

## 核心功能

### 1. 数据采集

**功能**：从多个电商和社交媒体平台采集真实数据

**支持平台**：
- 🛒 **京东**：商品信息、评论、销量
- 🛍️ **淘宝**：商品详情、用户评价
- 📱 **微博**：话题讨论、用户互动
- 📷 **小红书**：笔记内容、用户反馈

**合规性**：
- 遵守 robots.txt 规则
- 频率限制保护
- 数据去重和清洗

**使用技巧**：
- 关键词要精准，避免过于宽泛
- 采集数量适中，过多会影响性能
- 定期更新数据以保持时效性

### 2. 知识图谱

**功能**：自动构建商品和舆情的知识图谱

**图谱内容**：
- **实体**：商品、品牌、用户、话题
- **关系**：购买、评论、转发、点赞
- **属性**：价格、销量、情感倾向

**可视化**：
- 交互式图谱展示
- 实体关系探索
- 路径查询

### 3. 智能体仿真

**功能**：使用 OASIS 框架模拟用户行为

**仿真过程**：
1. 根据真实数据生成智能体
2. 设置初始状态和刺激
3. 多轮迭代模拟社交传播
4. 收集行为数据和指标

**输出指标**：
- 购买意愿变化
- 情感倾向分布
- 传播路径分析
- 影响力排名

### 4. 预测分析

**功能**：基于历史数据预测未来趋势

**预测模型**：
- **时序预测**：ARIMA、Prophet
- **因果推断**：DoWhy 框架
- **趋势分析**：移动平均、季节性分解

**预测内容**：
- 销量预测
- 舆情走势
- 用户增长
- 市场份额

### 5. 决策看板

**功能**：综合多维度数据生成决策建议

**决策维度**：
- 市场表现
- 用户反馈
- 竞品对比
- 风险评估

**证据链**：
- 每条结论都可追溯到原始数据
- 可靠性等级标注（强/一般/弱）
- 点击 Run chip 高亮原始 run

### 6. 分析报告

**功能**：自动生成专业分析报告

**报告内容**：
- 执行摘要
- 数据采集概况
- 仿真结果分析
- 预测趋势图表
- 决策建议
- 证据链与置信度

**报告格式**：
- HTML（自包含，可离线查看）
- PDF（适合打印）
- Markdown（适合编辑）

## 详细指南

### 项目管理

#### 创建项目

1. 点击右上角 **"新建项目"** 按钮
2. 填写项目信息：
   - **名称**：简洁明了，例如 "iPhone 15 舆情分析"
   - **描述**：说明分析目标和范围
3. 点击 **"确认"** 创建

#### 编辑项目

1. 在项目列表中找到目标项目
2. 点击 **"编辑"** 按钮
3. 修改项目信息
4. 点击 **"保存"**

#### 删除项目

1. 在项目列表中找到目标项目
2. 点击 **"删除"** 按钮
3. 确认删除操作

⚠️ **注意**：删除操作不可恢复，请谨慎操作。

### 材料上传

#### 支持的文件格式

- **PDF**：商品手册、营销方案
- **TXT**：纯文本文档
- **Markdown**：技术文档

#### 上传步骤

1. 进入项目详情页
2. 点击 **"上传文件"** 或拖拽文件到上传区域
3. 等待文件上传和处理
4. 查看自动生成的 Seed Report

#### 最佳实践

- 文件大小建议在 10MB 以内
- 内容要清晰、结构化
- 包含关键信息：商品特点、目标用户、市场定位

### 数据采集配置

#### 平台选择

根据分析目标选择合适的平台：

- **电商平台**（京东、淘宝）：适合商品销量和评价分析
- **社交媒体**（微博、小红书）：适合舆情和口碑分析

#### 关键词设置

**技巧**：
- 使用商品全称或品牌名
- 添加型号或系列名称
- 避免过于宽泛的词汇

**示例**：
- ✅ 好：`iPhone 15 Pro Max`
- ❌ 差：`手机`

#### 采集数量

**建议**：
- 小规模测试：100-200 条
- 常规分析：500-1000 条
- 深度分析：1000-5000 条

⚠️ **注意**：采集数量越多，处理时间越长。

### 仿真配置

#### 智能体数量

**建议**：
- 快速测试：1000
- 常规仿真：5000-10000
- 大规模仿真：50000-100000

#### 仿真轮次

**建议**：
- 快速测试：10 轮
- 常规仿真：20-50 轮
- 深度仿真：100+ 轮

#### 刺激强度

- **正面刺激**：模拟正面营销活动
- **负面刺激**：模拟负面舆情事件
- **中性刺激**：模拟自然传播

### 预测分析

#### 时序预测

**适用场景**：
- 销量预测
- 用户增长预测
- 趋势分析

**参数设置**：
- **预测周期**：建议 7-30 天
- **置信区间**：通常使用 95%

#### 因果推断

**适用场景**：
- 营销活动效果评估
- 价格变动影响分析
- 竞品对比分析

**注意事项**：
- 需要足够的历史数据
- 因果关系需要领域知识验证

### 决策看板使用

#### 查看决策

1. 进入项目详情页
2. 点击 **"决策看板"** 标签
3. 查看综合评分和建议

#### 证据链追溯

1. 点击任意决策结论
2. 查看支持该结论的证据
3. 点击 Run chip 高亮原始 run
4. 查看可靠性等级

#### 导出决策

1. 点击 **"导出"** 按钮
2. 选择格式：JSON、CSV、Markdown、HTML
3. 下载文件

## 常见问题

### Q1: 数据采集失败怎么办？

**可能原因**：
- 网络连接问题
- 平台反爬虫限制
- 关键词设置不当

**解决方案**：
1. 检查网络连接
2. 降低采集频率
3. 更换关键词重试
4. 查看错误日志获取详细信息

### Q2: 仿真运行很慢怎么办？

**可能原因**：
- 智能体数量过多
- 仿真轮次过多
- 系统资源不足

**解决方案**：
1. 减少智能体数量
2. 减少仿真轮次
3. 关闭其他占用资源的程序
4. 使用更高配置的机器

### Q3: 预测结果不准确怎么办？

**可能原因**：
- 历史数据不足
- 数据质量问题
- 模型参数不当

**解决方案**：
1. 采集更多历史数据
2. 检查数据质量和完整性
3. 调整模型参数
4. 尝试不同的预测模型

### Q4: 如何提高决策可靠性？

**建议**：
1. 采集更多真实数据
2. 增加仿真样本量
3. 使用多个预测模型交叉验证
4. 结合领域专家知识

### Q5: 报告生成失败怎么办？

**可能原因**：
- 数据不完整
- 系统资源不足
- 模板错误

**解决方案**：
1. 确保所有步骤都已完成
2. 检查系统资源
3. 重新生成报告
4. 查看错误日志

## 故障排查

### 前端问题

#### 页面无法加载

1. 检查浏览器控制台错误
2. 清除浏览器缓存
3. 尝试使用无痕模式
4. 更新浏览器到最新版本

#### 功能无响应

1. 检查网络连接
2. 查看浏览器控制台错误
3. 刷新页面重试
4. 联系技术支持

### 后端问题

#### API 请求失败

1. 检查后端服务是否运行
2. 查看后端日志
3. 检查 API 端点配置
4. 验证请求参数

#### 数据库错误

1. 检查数据库连接
2. 查看数据库日志
3. 验证数据完整性
4. 重启数据库服务
