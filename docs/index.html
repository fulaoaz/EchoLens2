Why EchoLens

这不是又一个数据大屏，而是一张“决策证据桌”。

电商团队面对的是评论碎片、竞品价格、KOL 传播和情绪波动的组合不确定性。EchoLens 的核心设计不是堆图表，而是把每一次采集、仿真、预测和建议都串成可复核证据。

Workflow

从材料上传到报告下载，一条线走完。

每个步骤都留下可追踪的运行记录，避免“看起来很合理，但无法复盘”的分析结论。

  1. 01项目创建

    录入商品、竞品、关键词和分析目标。

  2. 02材料与采集

    上传 PDF/TXT/Markdown，并采集评论与话题样本。

  3. 03图谱构建

    抽取品牌、卖点、KOL、情绪和证据片段。

  4. 04仿真预测

    用多智能体传播仿真和预测实验评估策略。

  5. 05决策看板

    汇总收益、风险、可靠性和关键路径。

  6. 06报告交付

    导出 HTML / Markdown，保留证据链与置信度。

Decision Studio

工作台像一张分析师桌面：左侧是材料，右侧是结论依据。

后端 Flask API 管理项目、采集、仿真、预测、决策和报告接口；前端 Vue 3 串联业务链路；Tauri 与 Capacitor 让桌面和移动客户端共享同一套能力。

Input商品材料 / 评论 / 话题

PDF、Markdown、TXT、电商与社媒样本。

EngineKuzu / LightRAG / OASIS

知识图谱、检索增强、多智能体传播仿真。

Output预测 / 决策 / 报告

可靠性等级、因果解释、证据链下载。
